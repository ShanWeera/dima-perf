## Results: Without Metadata

These tests analyze sequence diversity without parsing header metadata.

| Config | Command | Runtime | Output Size |
|--------|---------|---------|-------------|
| **Baseline** | `dima analyze -i input.fasta -o out.json` | **128.34s** | 29 MB |
| Binary (none) | `dima analyze -i input.fasta -O dima --compression 0 -o out.dima` | 135.37s | 5.5 MB |
| Binary (LZ4) | `dima analyze -i input.fasta -O dima -o out.dima` | 136.47s | 2.8 MB |
| Binary (Zstd) | `dima analyze -i input.fasta -O dima --compression 2 -o out.dima` | 135.11s | 2.4 MB |

**Key findings:**
- All configurations complete in ~128–136 seconds
- Binary format reduces output size by 80–93% with negligible runtime impact
- The performance bottleneck is computation, not I/O

## Results: With Metadata

Tests include header parsing and per-variant metadata aggregation using `--header-format "date|country|host"`.

| Config | Command | Runtime | Output Size |
|--------|---------|---------|-------------|
| **Baseline** | `dima analyze -i input.fasta -o out.json --header-format "..."` | 207.48s | 132 MB |
| **Optimized** | Same (columnar storage automatic) | **177.71s** | 132 MB |
| Binary (LZ4) | `dima analyze -i input.fasta -O dima -o out.dima --header-format "..."` | 178.55s | 15 MB |
| Binary (Zstd) | `dima analyze -i input.fasta -O dima --compression 2 -o out.dima --header-format "..."` | 209.38s | 8.6 MB |

**Key findings:**
- Metadata processing adds ~39% overhead (128s → 178s with columnar optimization)
- Columnar storage (automatic) provides 14–17% speedup for metadata workloads
- Binary + LZ4 reduces metadata output from 132 MB to 15 MB (89% smaller)

<Aside type="note">
Columnar storage is now enabled automatically when `--header-format` is specified. No separate flag is needed.
</Aside>

## Recommendations

| Use Case | Recommended Command |
|----------|---------------------|
| **Fastest (no metadata)** | `dima analyze -i input.fasta -o out.json` |
| **Smallest output** | `dima analyze -i input.fasta -O dima --compression 2 -o out.dima` |
| **With metadata (fast)** | `dima analyze -i input.fasta -o out.json --header-format "..."` |
| **With metadata (small)** | `dima analyze -i input.fasta -O dima -o out.dima --header-format "..."` |
