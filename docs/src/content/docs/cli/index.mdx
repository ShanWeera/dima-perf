## Conventions

DiMA follows established bioinformatics CLI conventions:

| Convention | Source | Example |
|-----------|--------|---------|
| `-i` / `-o` for input/output | samtools, bcftools | `dima analyze -i in.fasta -o out.json` |
| `-O` for output format | BCFtools `-O` flag | `dima analyze -O dima -o results.dima` |
| Format inferred from extension | samtools/bcftools | `-o results.tsv` → TSV auto-detected |
| `-` for stdin | Unix convention | `cat file.fasta \| dima analyze -i -` |
| `--no-header` for TSV | csvtk, tsv-utils | `dima view -O tsv --no-header` |

## Output Format Resolution

The output format is determined in this priority order:

1. **Explicit `-O` flag** — always wins
2. **Output file extension** — `.dima`, `.tsv`/`.tab`, `.jsonl`, else JSON
3. **Fallback** — JSON (pretty-printed to terminal, compact to pipe/file)

## Global Behavior

- **Progress bars** — Displayed on stderr when output is a terminal (suppressed with `-q`)
- **Color** — Disabled when `NO_COLOR=1` is set or output is not a terminal
- **Ctrl+C** — First press triggers graceful cancellation; second press forces immediate exit
- **Verbosity** — `-q` (quiet), default (warnings + progress), `-v` (info), `-vv` (debug), `-vvv` (trace)
